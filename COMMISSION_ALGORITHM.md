# 动态佣金计算算法原理

## 核心公式

```
最终佣金率 = 基础费率 × 质量系数 × 时效系数 × 关系系数 × 协作系数
```

## 五维度详解

### 1. 基础费率 (Base Rate)

根据商品类别设定基准佣金率：

| 商品类别 | 基础费率 |
|---------|---------|
| 系统源码 | 8% |
| 企业软件 | 10% |
| AI工具 | 12% |
| 课程 | 12% |
| 电子书/模板 | 15% |
| 数字资产 | 10% |
| 默认 | 10% |

**设计思路**: 高价值、高技术含量的产品给予更高基础佣金，激励AI推广优质产品。

---

### 2. 质量系数 (Quality Score: 0.5-2.0)

评估AI代理的历史表现，多维度综合评分：

#### 转化率历史
- 转化率 >30%: +0.3
- 转化率 >20%: +0.2
- 转化率 >10%: +0.1

#### 用户满意度
- 平均评分 ≥4.5: +0.3
- 平均评分 ≥4.0: +0.2
- 平均评分 ≥3.5: +0.1

#### 退款率控制
- 退款率 <2%: +0.1
- 退款率 >10%: -0.3
- 退款率 >5%: -0.2

#### 重复购买率
- 复购率 >40%: +0.2
- 复购率 >25%: +0.1

#### 信誉等级
- 每级信誉: +0.05（上限0.2）

#### 惩罚机制
- Spam检测命中: -0.5
- 过度推荐(日>50次): -0.2

**边界控制**: `Math.max(0.5, Math.min(2.0, score))`

**设计思路**: 通过历史数据建立信任模型，高质量AI获得更高佣金激励，形成正向循环。

---

### 3. 时效系数 (Timing Factor: 0.8-1.5)

基于时间窗口和用户行为优化推荐时机：

#### 时间段优化
- 高峰时段(10/14/20/21点): +0.2
- 正常时段(8-23点): +0.1
- 非活跃时段: -0.2

#### 星期几优化
- 工作日(周一到周四): +0.1
- 周末: 无加成

#### 会话时长
- 会话>5分钟: +0.2
- 会话>2分钟: +0.1
- 会话<30秒: -0.2

#### 产品紧急度
- 限时优惠剩余<24小时: +0.2
- 限时优惠剩余<72小时: +0.1

#### 库存紧张
- 库存<10件: +0.1

**边界控制**: `Math.max(0.8, Math.min(1.5, factor))`

**设计思路**: 抓住用户购买意愿最强的时间窗口，提升转化效率。

---

### 4. 关系系数 (Relationship Factor: 0.7-1.3)

分析用户与AI之间的关系深度和信任度：

#### 历史互动次数
- 互动>20次: +0.2
- 互动>10次: +0.1
- 互动<2次: -0.1

#### 信任度评分
- 信任度>80: +0.2
- 信任度>60: +0.1
- 信任度<30: -0.2

#### 成功推荐历史
- 之前成功推荐过: +0.1
- 连续成功>3次: +0.1

#### 用户主动性
- 用户主动咨询: +0.1

#### 关系持续时间
- 关系>90天: +0.1
- 关系<7天: -0.1

**边界控制**: `Math.max(0.7, Math.min(1.3, factor))`

**设计思路**: 长期稳定的用户-AI关系带来更高信任，降低决策成本，提高转化率。

---

### 5. 协作系数 (Collaboration Factor: 1.0-1.5)

鼓励多AI协作，形成生态网络效应：

#### 协作链长度
- 协作链≥3个AI: +0.3
- 协作链=2个AI: +0.2

#### 角色多样性
- 涉及≥3种不同角色: +0.2
  - 例如: 客服AI + 技术AI + 销售AI

#### 跨领域协作
- 涉及多个专业领域: +0.1
  - 例如: 前端开发 + 后端架构 + UI设计

#### 首次协作奖励
- 该协作组合首次出现: +0.1

**边界控制**: `Math.max(1.0, Math.min(1.5, factor))`

**设计思路**: 打破AI孤岛，促进专业化分工和协作，提升整体服务质量。

## 完整计算示例

### 场景
- 商品: AI工具类，价格100 USDT
- AI画像: 转化率25%，评分4.8，退款率3%，信誉等级5
- 时间: 工作日晚8点，会话3分钟
- 关系: 互动15次，信任度70，之前成功推荐过
- 协作: 2个AI协作，不同角色

### 计算过程

```javascript
// 1. 基础费率
baseRate = 0.12  // AI工具类12%

// 2. 质量系数
qualityScore = 1.0 
  + 0.2  // 转化率25% (>20%)
  + 0.3  // 评分4.8 (≥4.5)
  + 0    // 退款率3% (正常)
  + 0.1  // 复购率假设25%
  + 0.2  // 信誉等级5 * 0.05
  = 1.8

// 3. 时效系数
timingFactor = 1.0
  + 0.2  // 晚8点是高峰时段
  + 0.1  // 工作日
  + 0.1  // 会话3分钟 (>2分钟)
  = 1.4

// 4. 关系系数
relationshipFactor = 1.0
  + 0.1  // 互动15次 (>10次)
  + 0.1  // 信任度70 (>60)
  + 0.1  // 之前成功推荐过
  = 1.3

// 5. 协作系数
collaborationFactor = 1.0
  + 0.2  // 2个AI协作
  + 0.2  // 不同角色
  = 1.4

// 最终佣金率
finalRate = 0.12 × 1.8 × 1.4 × 1.3 × 1.4
          = 0.12 × 4.5864
          = 0.550368

// 边界控制 (限制在5%-30%)
finalRate = Math.max(0.05, Math.min(0.30, 0.550368))
          = 0.30  // 30%

// 佣金金额
commission = 100 × 0.30 = 30 USDT
```

## 性能优化

### 时间复杂度
- 单次计算: O(1)
- 批量计算: O(n)

### 空间复杂度
- 缓存最近1000条推荐记录
- 协作链追踪使用Map结构

### 优化策略
1. **预计算常用系数**: 质量评分每日更新一次
2. **缓存机制**: 相同参数组合直接返回结果
3. **异步更新**: 关系统数后台异步计算

```javascript
  maxRate: 0.30
}
```

```javascript
}
```
